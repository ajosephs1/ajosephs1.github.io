How to ask questions the general guide to netiquette

Before asking a technical question by e-mail, or in a newsgroup, or on a website chat board, do the following:

  1. Try to find an answer by searching the archives of the forum or mailing list you plan to post to.
  2. Try to find an answer by searching the Web.
  3. Try to find an answer by reading the manual.
  4. Try to find an answer by reading a FAQ.
  5. Try to find an answer by inspection or experimentation.
  6. Try to find an answer by asking a skilled friend.
  7. If you're a programmer, try to find an answer by reading the source code.
How the web works

computers connected to web are called clients and servers

clients: typical web user’s internet connected device. Example computer connected to wi-fi or phone connected to mobile network

servers: store, webpages, websites and apps and when a client wants to access websites a copy of the webpages is downloaded from the server

internet connection: allows you to send a receive data on the web (road)TCP/IP: Transmission control protocol & Internet Protocol are communication protocols that define how data should travel across the internet.

DNS: Domain name servers are like the address book for the website finding which server the site is stored on. These were invented to provide an easier reference than the IP address which are special numbers that look like this: 63.245.215.20.

HTTP: Hypertext Transfer Protocol is an application protocol that defines a language for clients and servers to speak to each other.

Component files: parts of the website that come in two different types- codefiles: websites are usually built form HTML, CSS and JavaScript- Assets: a collective name for all the other stuff that make up a website, such as images, music, video word documents and PDFs.

  • HTML: Hyper Text Markup Language
  • Hypertext: the content of webpage
  • Webpage: tags used to define how content will be displayed
  • Language: syntax

ID and Classes

ID should be used only once on a page
The id style adds and potentially overrides element style

Classes can be used to style multiple elements with the class id
However certain elements within the class can be tagged and styled differently ex:
<h1 class=”yael”>A Heading</h1> Will not be styled by : p.yael{background color: yellow}
but will be styled by: .yeal{background-color yellow}

  • Use <div> to  style whole section of a page and is used as a tag rather than attribute <div></div>. It is a block line container
  • <span> is an inline container therefore you can use within tags. Add class attribute for example “worlboss”. Then use css: span.worlboss {whatever style you want}
Positioning

  • postion: relative allows you to move an element relative from the position of its normal flow position within the document and relative to its parent container

  • position: absolute moves the element from the normal flow of the document and will offset the document to the nearest positioned anchor
  • position: fixed is similar to absolute whereas it is removed from the normal flow and other elements won’t recognize it the only difference is that it remains fixed when scrolling (good for navigation bars)
  • z-indexing controls the layer position of an overlapping element: by default the element coming later in the HTML mark-up will appear on top
  • another way of moving elements is by using margin: auto to move items to the middle of the page horizontally

Pseudo classes

pseudo classes can be used within a style selector to add a function to a class

  • an example would be p:hover {transform: scale(2.1);} this would change the size of the paragraph to 1.2x its original size when hovering
Applied visual design

The hsl() color option makes it easy to change the tone of a colour once you find a hue that you like

  • the h: represents hue and is represented by its angular number on a colour wheel; red=0 and green=120
  • s: represents saturation and is measured in percentage the less saturated the percentage the more grey added
  • l: represents lightness and is also represented in percentage 0% being white and 100% being black

- Gradual CSS Linear Gradient: general syntax angle of gradient, transition colours left to right background: linear-gradient(90deg, red, yellow, rgb(204, 204, 255));

    • Responsive web design principles

    Media Queries are a new technique introduced in CSS3 that change the presentation of content based on different viewport sizes. The viewport is a user's visible area of a web page, and is different depending on the device used to access the site.

    • the syntax for a media query: @media (min-height: 350px) { /* CSS Rules */ }
    • Making images responsive with CSS is actually very simple. You just need to add these properties to an image: img {max-width: 100%; height: auto;}
